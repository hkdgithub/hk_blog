暂无

问题和解决方案 1.用户提交的信息不管格式是不是正确的都会进行提交
原来：
input type="submit" value="登陆"onclick="infoSubmit()"
现在：
form action="/" onsubmit="return infoSumit()"
2.使用了分页插件，但总是不能分页
原来：
groupId: com.github.pagehelper groupId     artifactId: pagehelper     version:4.1.6
现在：
groupId: com.github.pagehelper     artifactId:pagehelper-spring-boot-starter     version:1.2.5
3.java.lang.NullPointerException: null
原来：method="get"
现在：method="post"
4.onsubmit="return writeBlogSubmit()"不起作用
原来：上传博客预览图：
  id="blogCoverImage" name="file" type="file" id="txt"
现在：上传博客预览图：
  id="blogCoverImage" name="file" type="file"
5.textarea使用火狐浏览器不能换行
在经过多个浏览器测试后，发现是浏览器兼容有问题，有的能换行，有的不能换行
6.404 找不到路径
出现了很多次，有时是没写@Controller，有时是页面路径上设置错误
7.表中自增长的数据删除后，再次添加不是从1开始
解决的方法：truncate table table_name
8.e: java.sql.SQLException: Incorrect string value: '\xE5\x85\xA5\xE9\x97\xA8...' for column 'blogContent' at row 1
解决的方法：将字段的编码方式改为了utf8mb4
9.java.lang.IllegalArgumentException: Request header is too large.
问题：请求的文本长度太长
解决方法：application.properties中配置server.max-http-header-size=10000000
10.发送邮件出现异常org.springframework.mail.MailAuthenticationException: Authentication failed; nested exception is javax.mail.AuthenticationFailedException: 535 Login fail. Authorization code is expired
解决的方法：发现是授权码过期。重新取qq邮箱申请了一个授权码
11.在后台撰写内容时的格式，在前台不能被转义
解决的方法：使用th:utext
12.表格中的一列是按钮，点击按钮弹出bootstarp拟态框，拟态框的标签获得按钮所在行的值，并且带格式
解决的办法：看生活分享->文章分享
但是遇到了一个始终不能解决的问题，我弄了好久，终于能带格式了，但是，博客内容必须要在表中已经遍历出来，我使用 多种隐藏标签的方法，一旦隐藏，js中就获取不了，或者不带格式，后来就没有弄了，所以现在看到的后台还是不能在修改时有一个预览博客内容的界面
13.使用拦截器的时候，静态资源也被拦截了
在这里也碰壁了，在网上查了很多资料，我拦截器的配置是这样的
      					// 这里的addPathPatterns("/**")为配置需要拦截的方法“/**”代表所有，而后excludePathPatterns("/enterAdminLogin")等方法为排除哪些方法不进行拦截
      					registry.addInterceptor(new interceptor()).addPathPatterns("/**").excludePathPatterns("/enterAdminLogin").excludePathPatterns
              			 ("/enterUserLogin").excludePathPatterns("/enterUserRegister").excludePathPatterns("static/**");
      					super.addInterceptors(registry);
这样是可以已经放行了静态资源，但是始终被过滤，后来发现不是这个的问题。
14.其实最大的问题不是这些报错，是需求不明确，很多地方修修补补，浪费了大量的时间，所以我建议一定要明确需求，不然会浪费很多精力。
总结与感想 也没有什么想说的，只是觉的，不足的地方还有很多，比如拿这个项目来说吧，我很多地方都可以使用更好的东西，但是为了加快进度，就没有使用，如可以加driud连接池，动态sql，redis，jq等等，但是由于不熟练的缘故，就放弃了，可以看出我知识不牢固。 加油吧！
